# پلن ساخت فروشگاه آرایشی و مراقبت از پوست

این یک پروژه بزرگ است. آن را در چند فاز تحویل می‌دهم تا هر فاز قابل تست باشد، اما خروجی نهایی کامل و عملیاتی خواهد بود (بدون Placeholder).

## نکات مهم درباره تکنولوژی

پروژه شما روی **TanStack Start + React + TypeScript + Tailwind v4** اجرا می‌شود (نه Next.js). این استک از نظر قابلیت معادل Next.js است: SSR، روتینگ فایل‌محور، Server Functions، SEO کامل. تمام نیازهای شما را پوشش می‌دهد.

برای بک‌اند **Lovable Cloud** (مبتنی بر Postgres + Auth + Storage) فعال می‌شود — بدون نیاز به اکانت جداگانه.

نکته درباره **Dark Mode** و **PWA**: طبق راهنمای داخلی، در نسخه اولیه dark mode اضافه نمی‌کنیم تا تمرکز روی کیفیت طراحی صورتی/لوکس باشد؛ بعداً قابل افزودن است. PWA پایه (manifest + meta) اضافه می‌شود.

## فاز ۱ — طراحی و اسکلت (این مرحله)
- دیزاین سیستم فارسی RTL: فونت Vazirmatn، پالت صورتی ملایم/سفید/خاکستری روشن، glassmorphism، گوشه‌های گرد، انیمیشن نرم
- ساختار راست‌چین در `__root.tsx` (`dir="rtl"`, `lang="fa"`)
- هدر، فوتر، ناوبری، جستجو
- صفحه اصلی کامل: اسلایدر، دسته‌بندی، محصولات ویژه/جدید/پرفروش، تخفیف‌ها، برندها، مقالات، نظرات، FAQ، خبرنامه

## فاز ۲ — کاتالوگ محصول
- صفحات: لیست محصولات، دسته، برند، جستجو، فیلتر/مرتب‌سازی
- صفحه محصول کامل (گالری با زوم، مشخصات، ترکیبات، روش مصرف، امتیاز، نظرات، محصولات مشابه/مکمل)
- سبد خرید و علاقه‌مندی‌ها (Cloud)

## فاز ۳ — احراز هویت و سفارش
- ثبت‌نام/ورود ایمیل-رمز فارسی (Lovable Cloud Auth)
- پنل کاربر: سفارش‌ها، رسیدها، کد رهگیری، اطلاعات شخصی، تغییر رمز، علاقه‌مندی‌ها
- چک‌اوت → کارت‌به‌کارت: نمایش شماره کارت/بانک/صاحب حساب + دکمه کپی، آپلود رسید، شماره پیگیری، مبلغ، توضیحات
- وضعیت اولیه: «در انتظار بررسی پرداخت»

## فاز ۴ — پنل مدیریت
- نقش `admin` با جدول `user_roles` و تابع `has_role` (طبق الزامات امنیتی)
- مدیریت سفارش‌ها (مشاهده رسید با بزرگنمایی، تایید/رد، علت رد، تغییر وضعیت‌ها)
- ارسال: شرکت حمل، کد رهگیری، توضیح
- CRUD: محصولات (چند تصویر، ویژگی‌ها)، برند، دسته، اسلایدر، بنر، صفحات، وبلاگ، FAQ
- گزارش‌ها: فروش روزانه/ماهانه/سالانه، پرفروش‌ها، مشتریان برتر، موجودی

## فاز ۵ — SEO و امنیت نهایی
- متاتگ‌های فارسی، Open Graph، Twitter، JSON-LD (Product, Organization, BreadcrumbList)
- `sitemap.xml` داینامیک (شامل محصولات/دسته‌ها/مقالات) + `robots.txt`
- اعتبارسنجی ورودی با Zod، RLS کامل روی همه جداول، آپلود امن تصاویر، حفاظت از مسیرهای ادمین
- بهینه‌سازی سرعت، lazy loading، PWA manifest

## جزئیات فنی (برای مرجع)
- **روتینگ**: TanStack Router فایل‌محور زیر `src/routes/` — مسیرهای عمومی در ریشه، پنل کاربر/ادمین زیر `_authenticated/`
- **دیتابیس**: جداول `products, categories, brands, product_images, product_attributes, cart_items, wishlist, orders, order_items, payments, shipments, addresses, reviews, coupons, slides, banners, pages, blog_posts, faqs, newsletter, user_roles, profiles, bank_accounts` — همگی با RLS و GRANT صحیح
- **استوریج**: باکت‌های `product-images` (عمومی)، `receipts` (خصوصی)، `blog-images` (عمومی)
- **Server Functions**: تمام نوشتن‌های ادمین با `requireSupabaseAuth` + بررسی `has_role('admin')`
- **پرداخت**: بدون درگاه؛ جدول `bank_accounts` برای کارت‌به‌کارت قابل مدیریت توسط ادمین

## سؤال قبل از شروع

برای بهترین کیفیت طراحی، لطفاً یک نمونه برند مرجع که از نظر حس‌وحال نزدیک به سلیقه شماست بگویید (مثلاً Sephora, Glossier, Charlotte Tilbury, Rare Beauty، یا یک برند ایرانی). اگر ترجیحی ندارید، طراحی الهام‌گرفته از **Glossier × Charlotte Tilbury** (مینیمال صورتی لوکس) اجرا می‌کنم.

پس از تایید پلن، از فاز ۱ شروع می‌کنم.
